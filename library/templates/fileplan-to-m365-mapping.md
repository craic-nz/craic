# File Plan to Microsoft 365 Mapping

> A planning table to map a corporate file plan or records classification to a Microsoft 365 structure, including retention and sensitivity labels.

- **Purpose:** To plan and document how each function and sub-function in the agency's file plan or records classification will be represented in Microsoft 365, including where records will live (sites and libraries) and which retention and sensitivity labels apply. This keeps recordkeeping and information protection aligned as the agency moves information into a collaboration platform.
- **Owner / completed by:** [RECORDS MANAGER] (with [INFORMATION SECURITY] and [SYSTEM ADMINISTRATOR])
- **Related legislation / standards:**
  - Public Records Act 2005 and Archives New Zealand recordkeeping guidance: https://www.archives.govt.nz/manage-information
  - Privacy Act 2020 (protection of personal information): https://www.privacy.org.nz/privacy-act-2020/
  - New Zealand Information Security Manual (classification and handling): https://www.nzism.gcsb.govt.nz/

## How to use this template

- Start from your existing file plan or records classification, and map one row per function or sub-function.
- Decide the proposed Microsoft 365 location for each (for example, a SharePoint site and document library), keeping the structure aligned to business functions rather than team names where possible.
- Assign a retention label that reflects the relevant item in your retention and disposal schedule, so disposal is driven by policy rather than manual action.
- Assign a sensitivity label that reflects the security classification of the information, so protection travels with the content.
- Use the notes column to capture decisions, dependencies, and any records that should not be moved (for example, records subject to a legal hold or pending transfer).

## Notes

- Keep retention labels mapped back to approved retention and disposal schedule items so that disposal in the platform remains authorised.
- Sensitivity labels should mirror the agency's security classification scheme so that protection and handling are consistent across systems.
- Migration into a collaboration platform is a good point to appraise content: not everything needs to move, and some records may be due for disposal or transfer first.
- This template is vendor-neutral in intent; product and label names will depend on how your agency has configured its environment.
- This is a general template to be adapted to your agency's policies, file plan and platform configuration; it is not legal advice.
